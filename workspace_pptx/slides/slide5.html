04 // PRINCIPLE

HARNESS ENGINEERING

Stop tuning the model. Engineer the harness.

The model is fixed. The scaffolding around it is where reliability and steerability come from.

SEPARATION

Decouple the LLM from the orchestration logic — swap models without rewriting agents.

REUSABILITY

Same harness, different agents and tools. Subagents become composable units.

STEERABILITY

Explicit planning beats implicit reasoning. Inspectable TODOs over chain-of-thought.

FIVE HARNESS LAYERS

CONTEXT

Keep main agent context lean — only essentials reach the orchestrator.

QUARANTINE

Each subagent runs in its own context window — no cross-contamination.

OFFLOAD

Large tool results land in the virtual FS, not the chat history.

PLAN

write_todos forces explicit task decomposition before execution.

PERSIST

Long-term memory + project artifacts outside the conversation state.
